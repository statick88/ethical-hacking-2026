28.1 Introducción a Máquinas Virtuales en Seguridad

Las máquinas virtuales son esenciales en seguridad informática para crear entornos aislados, reproducibles y portátiles para pruebas de pentesting, análisis de malware y laboratorios de aprendizaje.

28.1.1 Objetivos del Minicurso

  • Configurar VMware Fusion y VirtualBox para laboratorios de seguridad
  • Instalar y configurar Kali Linux como máquina virtual
  • Optimizar el rendimiento y seguridad de las VMs
  • Crear laboratorios de pentesting con VMs
  • Automatizar despliegues de laboratorios

28.2 1. Conceptos Fundamentales

28.2.1 1.1 Virtualización

La virtualización permite ejecutar múltiples sistemas operativos en un mismo hardware mediante un hipervisor.

Tipos de hipervisores: - Tipo 1 (Bare-metal): Ejecuta directamente sobre hardware (VMware ESXi, Hyper-V) - Tipo 2 (Hosted): Ejecuta sobre sistema operativo host (VMware Fusion, VirtualBox)

28.2.2 1.2 Ventajas en Seguridad Informática

  • Aislamiento: Malware no afecta sistema host
  • Portabilidad: Laboratorios fácilmente compartibles
  • Snapshots: Puntos de restauración rápidos
  • Clonación: Múltiples entornos idénticos
  • Redes aisladas: Entornos de prueba controlados

28.5 4. Instalación de Kali Linux

28.5.1 4.1 Descarga de Imagen Oficial

# Opciones de descarga:
# 1. Image: https://www.kali.org/get-kali/#kali-virtual-machines
# 2. Torrent: Más rápido para descargas grandes
# 3. Mirrors: https://http.kali.org/

# Verificar integridad (importante para seguridad)
sha256sum kali-linux-2026.1-vmware-amd64.7z
# Comparar con hash oficial en https://www.kali.org/get-kali/

28.5.2 4.2 Instalación en VMware Fusion

Paso 1: Crear máquina virtual

# VMware Fusion > File > New...
# Select "Import an existing virtual machine"
# Seleccionar archivo .vmx o .ovf de Kali

Paso 2: Configurar hardware

- Procesadores: 2+ núcleos
- RAM: 2-4 GB (mínimo 2 GB)
- Disco: 40-60 GB (dinámico)
- Red: NAT o Host-Only (según necesidad)
- USB: Habilitar dispositivos USB

Paso 3: Primer arranque

# Usuario por defecto: kali
# Contraseña por defecto: kali

# Cambiar contraseña inmediatamente
sudo passwd kali

28.5.3 4.3 Instalación en VirtualBox

Paso 1: Importar OVA/OVF

# VirtualBox > File > Import Appliance
# Seleccionar archivo .ova de Kali

Paso 2: Configurar máquina

# Configuración > Sistema
- Placa base: Linux 64-bit
- RAM: 2048 MB
- Procesadores: 2

# Configuración > Pantalla
- Memoria de video: 128 MB
- Aceleración 3D: Habilitar

# Configuración > Red
- Adapter 1: NAT o Host-Only

Paso 3: Configurar VirtualBox Guest Additions

# Dentro de Kali Linux
sudo apt update
sudo apt install -y build-essential dkms linux-headers-$(uname -r)

# Insertar Guest Additions CD
# VirtualBox > Devices > Insert Guest Additions CD image

# Montar y ejecutar
sudo mkdir -p /mnt/cdrom
sudo mount /dev/cdrom /mnt/cdrom
cd /mnt/cdrom
sudo ./VBoxLinuxAdditions.run

# Reiniciar
sudo reboot

28.6.2 5.2 Configuración de Red

# Ver interfaces de red

# Configurar IP estática (ejemplo)

Ejemplo de configuración estática:

network:
  version: 2
  ethernets:
    eth0:
      dhcp4: no
      addresses: [192.168.56.10/24]
      gateway4: 192.168.56.1
      nameservers:

Aplicar cambios:

sudo netplan apply

28.8 7. Gestión de Snapshots y Clonación

28.8.1 7.1 Snapshots en VMware Fusion

Crear snapshot:

# Desde interfaz gráfica
# Virtual Machine > Snapshots > Take Snapshot

# Desde línea de comandos
vmrun snapshot "ruta/vm.vmx" "Snapshot antes de explotación"

Gestionar snapshots:

# Listar snapshots
vmrun listSnapshots "ruta/vm.vmx"

# Revertir a snapshot
vmrun revertToSnapshot "ruta/vm.vmx" "Snapshot nombre"

# Eliminar snapshot
vmrun deleteSnapshot "ruta/vm.vmx" "Snapshot nombre"

28.8.2 7.2 Snapshots en VirtualBox

Crear snapshot:

# Desde interfaz gráfica
# Machine > Take Snapshot

# Desde línea de comandos
VBoxManage snapshot "NombreVM" take "Snapshot1" --description "Estado inicial"

Gestionar snapshots:

# Listar snapshots
VBoxManage snapshot "NombreVM" list

# Revertir a snapshot
VBoxManage snapshot "NombreVM" restore "Snapshot1"

# Eliminar snapshot
VBoxManage snapshot "NombreVM" delete "Snapshot1"

28.8.3 7.3 Clonación de Máquinas

VMware Fusion:

# Interfaz gráfica
# Virtual Machine > Clone

# Línea de comandos
vmrun clone "original.vmx" "clon.vmx" linked

VirtualBox:

# Interfaz gráfica
# Machine > Clone

# Línea de comandos
VBoxManage clonevm "OriginalVM" --name "ClonVM" --register

28.9 8. Automatización con Vagrant

28.9.1 8.1 Introducción a Vagrant

Vagrant permite automatizar la creación y configuración de máquinas virtuales.

28.9.2 8.2 Instalación

# macOS con Homebrew
brew install vagrant

# Verificar instalación
vagrant --version

28.9.3 8.3 Vagrantfile para Kali Linux

# Vagrantfile
Vagrant.configure("2") do |config|
  
  # Kali Linux
  config.vm.define "kali" do |kali|
    kali.vm.box = "kalilinux/rolling"
    kali.vm.hostname = "kali"
    
    # Red host-only
    kali.vm.network "private_network", ip: "192.168.56.10"
    
    # Configuración de recursos
    kali.vm.provider "virtualbox" do |vb|
      vb.name = "Kali-Linux-Pentest"
      vb.memory = "4096"
      vb.cpus = 2
      vb.gui = true
    end
    
    # Provisionamiento
    kali.vm.provision "shell", inline: <<-SHELL
      apt update
      apt full-upgrade -y
      apt autoremove -y
      
      # Instalar herramientas adicionales
      apt install -y gobuster impacket-scripts
      
      # Configurar SSH
      systemctl enable ssh
      systemctl start ssh
    SHELL
  end
  
  # Máquina vulnerable (Metasploitable)
  config.vm.define "metasploitable" do |meta|
    meta.vm.box = "metasploitable2-linux"
    meta.vm.hostname = "metasploitable"
    meta.vm.network "private_network", ip: "192.168.56.20"
    
    meta.vm.provider "virtualbox" do |vb|
      vb.name = "Metasploitable2"
      vb.memory = "1024"
      vb.cpus = 1
      vb.gui = false
    end
  end
end

# Detener laboratorio

# Reconfigurar

28.12.2 11.2 Script de Automatización

#!/bin/bash
# setup-pentest-lab.sh

# Configuración
VM_NAME="Kali-Pentest-Lab"
VM_MEMORY="4096"
VM_CPUS="2"
NETWORK="192.168.56.0/24"

# Función para crear VM
create_vm() {
    echo "Creando máquina virtual..."
    
    # Usar Vagrant para crear VM
    vagrant up
    
    # O alternativamente con VirtualBox
    # VBoxManage createvm --name "$VM_NAME" --ostype "Linux_64" --register
    # VBoxManage modifyvm "$VM_NAME" --memory "$VM_MEMORY" --cpus "$VM_CPUS"
}

# Función para configurar red
setup_network() {
    echo "Configurando red..."
    
    # Configurar IP estática en Kali
    vagrant ssh kali -c "sudo sed -i 's/dhcp/static/' /etc/netplan/01-netcfg.yaml"
    vagrant ssh kali -c "echo 'addresses: [192.168.56.10/24]' >> /etc/netplan/01-netcfg.yaml"
    vagrant ssh kali -c "sudo netplan apply"
}

# Función para instalar herramientas
install_tools() {
    echo "Instalando herramientas de pentesting..."
    
    vagrant ssh kali -c "sudo apt update && sudo apt install -y nmap nikto sqlmap gobuster"
}

# Función para crear snapshot
create_snapshot() {
    echo "Creando snapshot inicial..."
    VBoxManage snapshot "$VM_NAME" take "Initial-Setup" --description "Estado inicial del laboratorio"
}

# Menú principal
case "$1" in
    create)
        create_vm
        setup_network
        install_tools
        create_snapshot
        ;;
    start)
        vagrant up
        ;;
    stop)
        vagrant halt
        ;;
    destroy)
        vagrant destroy -f
        ;;
    *)
        echo "Uso: $0 {create|start|stop|destroy}"
        exit 1
        ;;
esac
