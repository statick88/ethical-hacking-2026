18  Quiz Unidad 1: Fundamentos y Reconocimiento Agéntico

Puntaje mínimo: 7/10 (70%)

18.1 Preguntas de Opción Múltiple

18.1.1 Pregunta 1 (Básica)

¿Cuál es la primera fase del pentesting según la metodología PTES?

  1. Explotación
  2. Reconocimiento
  3. Reporte
  4. Escaneo

Respuesta correcta: b Explicación: El reconocimiento (Intelligence Gathering) es la primera fase donde se recolecta información sobre el objetivo.

18.1.2 Pregunta 2 (Básica)

¿Qué tipo de reconocimiento NO toca directamente el objetivo?

  1. Escaneo de puertos
  2. OSINT pasivo
  3. Nmap activo
  4. Conexión directa

Respuesta correcta: b Explicación: El OSINT pasivo usa fuentes públicas sin interactuar con el objetivo.

18.1.3 Pregunta 3 (Intermedia)

¿Cuál de las siguientes herramientas se utiliza principalmente para reconocimiento web automatizado?

  1. Nmap
  2. Metasploit
  3. Recon-ng
  4. Burp Suite

Respuesta correcta: c Explicación: Recon-ng es un framework modular específico para reconocimiento web.

18.1.4 Pregunta 4 (Intermedia)

¿Qué flag de Nmap realiza un escaneo SYN (half-open)?

  1. -sT
  2. -sU
  3. -sS
  4. -sP

Respuesta correcta: c Explicación: -sS es el escaneo SYN que no completa el handshake TCP.

18.1.5 Pregunta 5 (Intermedia)

¿Cuál es el propósito del parámetro -sC en Nmap?

  1. Escaneo de servicios
  2. Ejecución de scripts NSE por defecto
  3. Detección de SO
  4. Escaneo rápido

Respuesta correcta: b Explicación: -sC ejecuta los scripts por defecto del NSE.

18.1.6 Pregunta 6 (Avanzada)

En el contexto de OSINT, ¿qué es un “Google Dork”?

  1. Un exploit para Google
  2. Búsqueda avanzada con operadores
  3. Una herramienta de hacking
  4. Un tipo de ataque

Respuesta correcta: b Explicación: Los Google Dorks son consultas avanzadas usando operadores como site:, filetype:, etc.

18.1.7 Pregunta 7 (Básica)

¿Qué protocolo usa Nmap por defecto para el escaneo?

  1. UDP
  2. TCP
  3. ICMP
  4. HTTP

Respuesta correcta: b Explicación: Nmap escanea puertos TCP por defecto.

18.1.8 Pregunta 8 (Intermedia)

¿Cuál es el rango de puertos “bien conocidos” en TCP?

  1. 0-1023
  2. 1024-49151
  3. 49152-65535
  4. 1-80

Respuesta correcta: a Explicación: Los puertos 0-1023 son puertos bien conocidos (well-known ports).

18.1.9 Pregunta 9 (Avanzada)

¿Qué herramienta es mejor para enumeración masiva de subdominios?

  1. Nmap básico
  2. theHarvester
  3. Burp Suite
  4. SQLmap

Respuesta correcta: b Explicación: theHarvester está diseñado para enumerar subdominios, emails y más.

18.1.10 Pregunta 10 (Intermedia)

¿Cuál es la principal ventaja del escaneo UDP comparado con TCP?

  1. Más rápido
  2. Más silencioso
  3. Puede detectar servicios sin handshake
  4. Siempre más detallado

Respuesta correcta: c Explicación: El escaneo UDP puede identificar servicios que responden sin necesidad de handshake.

18.2 Escenarios Prácticos

18.2.1 Escenario 1

Situación: Un pentester recibe un objetivo: empresa.com. Necesita identificar todos los subdominios posibles.

Pregunta: ¿Cuál es el mejor enfoque inicial?

  1. Ejecutar nmap -p- empresa.com
  2. Usar Google Dorks para buscar subdominios
  3. Enviar emails de phishing
  4. Atacar directamente el DNS

Respuesta esperada: Usar Google Dorks como site:empresa.com -www y herramientas como subfinder o amass.

Criterios: El estudiante debe identificar que OSINT es la primera opción antes de escaneo activo.

18.2.2 Escenario 2

Situación: Durante un escaneo con Nmap, el resultado muestra “filtered” en varios puertos.

Pregunta: ¿Qué significa el estado “filtered”?

  1. El puerto está cerrado
  2. El puerto está abierto
  3. Probablemente hay un firewall filtrando
  4. El escaneo falló

Respuesta esperada: “Filtered” indica que un firewall o filtro de red está evitando que Nmap determine el estado del puerto.

Criterios: El estudiante debe entender la diferencia entre estados: open, closed, filtered.
