Introducción a Docker

Docker es una plataforma de contenerización que permite empaquetar aplicaciones y sus dependencias en contenedores portables. En seguridad informática, Docker es esencial para crear entornos aislados para pruebas, laboratorios y herramientas de pentesting.

Objetivos del Minicurso

  • Comprender los conceptos fundamentales de contenerización
  • Aprender a usar Docker para laboratorios de seguridad
  • Crear y gestionar contenedores personalizados
  • Automatizar despliegues con Docker Compose
  • Integrar Docker en flujos de trabajo de pentesting

1. Conceptos Fundamentales

1.1 ¿Qué es un Contenedor?

Un contenedor es una instancia aislada de un sistema operativo que comparte el kernel del sistema anfitrión pero tiene su propio sistema de archivos, procesos y red.

Diferencias clave: - Máquina Virtual: Incluye todo el sistema operativo guest (pesado) - Contenedor: Solo incluye la aplicación y sus dependencias (ligero)

1.2 Arquitectura Docker

O en formato ASCII:

┌─────────────────────────────────────┐
│         Aplicación                  │
├─────────────────────────────────────┤
│      Librerías / Binarios           │
├─────────────────────────────────────┤
│        Contenedor                   │
├─────────────────────────────────────┤
│      Kernel del Host                │
└─────────────────────────────────────┘

2. Instalación y Configuración

2.1 Instalación en Linux (Kali/Debian/Ubuntu)

#| label: fig-install-docker
#| fig-cap: "Comandos para instalar Docker en Kali/Debian"
#| echo: true

# Actualizar repositorios
sudo apt update

# Instalar dependencias
sudo apt install -y apt-transport-https ca-certificates curl gnupg lsb-release

# Añadir clave GPG de Docker
curl -fsSL https://download.docker.com/linux/debian/gpg | sudo gpg --dearmor -o /usr/share/keyrings/docker-archive-keyring.gpg

# Añadir repositorio
echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/docker-archive-keyring.gpg] https://download.docker.com/linux/debian $(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

# Instalar Docker
sudo apt update
sudo apt install -y docker-ce docker-ce-cli containerd.io

# Añadir usuario al grupo docker
sudo usermod -aG docker $USER

# Probar instalación
docker --version
docker run hello-world

2.2 Configuración Básica

# Verificar estado de Docker
sudo systemctl status docker

# Iniciar Docker al arrancar
sudo systemctl enable docker

# Configurar daemon (opcional)
sudo nano /etc/docker/daemon.json

Ejemplo de configuración:

{
  "log-driver": "json-file",
  "log-opts": {
    "max-size": "10m",
    "max-file": "3"
  },
  "storage-driver": "overlay2"
}

3. Comandos Básicos de Docker

3.1 Gestión de Imágenes

# Listar imágenes locales
docker images
docker image ls

# Buscar imágenes en Docker Hub
docker search kali-linux
docker search pentest

# Descargar una imagen
docker pull kalilinux/kali-rolling
docker pull ubuntu:22.04

# Eliminar una imagen
docker rmi <imagen_id>
docker image rm <imagen_id>

# Etiquetar una imagen
docker tag mi-imagen:latest usuario/mi-imagen:v1.0

3.2 Gestión de Contenedores

# Ejecutar contenedor interactivo
docker run -it kalilinux/kali-rolling /bin/bash

# Ejecutar contenedor en segundo plano
docker run -d --name mi-contenedor ubuntu:22.04 sleep 3600

# Listar contenedores activos
docker ps

# Listar todos los contenedores (incluyendo detenidos)
docker ps -a

# Detener contenedor
docker stop <container_id>

# Iniciar contenedor detenido
docker start <container_id>

# Reiniciar contenedor
docker restart <container_id>

# Eliminar contenedor
docker rm <container_id>
docker rm -f <container_id>  # Forzar eliminación

# Ver logs de contenedor
docker logs <container_id>
docker logs -f <container_id>  # Seguir logs en tiempo real

# Ejecutar comando en contenedor activo
docker exec -it <container_id> /bin/bash

4. Opciones de Red y Almacenamiento

4.1 Redes Docker

# Listar redes
docker network ls

# Crear red personalizada
docker network create mi-red

# Conectar contenedor a red
docker run -d --name contenedor1 --network mi-red ubuntu:22.04
docker run -d --name contenedor2 --network mi-red ubuntu:22.04

# Conectar contenedor existente a red
docker network connect mi-red contenedor_existente

# Desconectar contenedor de red
docker network disconnect mi-red contenedor_existente

# Ver detalles de red
docker network inspect mi-red

Tipos de redes: - bridge: Red por defecto (aislada) - host: Usa red del host - none: Sin red - overlay: Para clústeres (Swarm) - macvlan: Asigna MAC address al contenedor

4.2 Volúmenes y Directorios Montados

# Montar directorio del host en contenedor
docker run -it -v /ruta/host:/ruta/contenedor ubuntu:22.04

# Montar volumen con nombre
docker volume create mi-volumen
docker run -it -v mi-volumen:/ruta/contenedor ubuntu:22.04

# Montar como solo lectura
docker run -it -v /ruta/host:/ruta/contenedor:ro ubuntu:22.04

# Copiar archivos entre host y contenedor
docker cp /ruta/host/archivo.txt <container_id>:/ruta/contenedor/
docker cp <container_id>:/ruta/contenedor/archivo.txt /ruta/host/

5. Dockerfile: Creación de Imágenes Personalizadas

5.1 Estructura Básica de Dockerfile

# Imagen base
FROM kalilinux/kali-rolling:latest

# Maintainer (obsoleto, usar LABEL)
LABEL maintainer="diego@ejemplo.com"

# Variables de entorno
ENV DEBIAN_FRONTEND=noninteractive

# Actualizar e instalar paquetes
RUN apt update && apt install -y \
    nmap \
    nikto \
    sqlmap \
    && rm -rf /var/lib/apt/lists/*

# Directorio de trabajo
WORKDIR /app

# Copiar archivos locales al contenedor
COPY . /app

# Exponer puertos
EXPOSE 80 443 22

# Comando por defecto al iniciar
CMD ["/bin/bash"]

5.2 Ejemplo: Imagen de Pentesting

# Dockerfile para entorno de pentesting
FROM kalilinux/kali-rolling:latest

# Configurar entorno
ENV DEBIAN_FRONTEND=noninteractive
ENV LANG=C.UTF-8

# Actualizar sistema
RUN apt update && apt upgrade -y

# Instalar herramientas de pentesting
RUN apt install -y \
    # Herramientas de enumeración
    nmap \
    enum4linux \
    ldap-utils \
    # Herramientas web
    nikto \
    dirb \
    dirbuster \
    wfuzz \
    sqlmap \
    # Herramientas de explotación
    metasploit-framework \
    exploitdb \
    # Herramientas de post-explotación \
    mimikatz \
    # Herramientas de red \
    wireshark \
    tcpdump \
    netcat-traditional \
    # Herramientas de fuerza bruta \
    hydra \
    john \
    # Limpieza \
    && rm -rf /var/lib/apt/lists/*

# Crear usuario no root
RUN useradd -m -s /bin/bash pentester
USER pentester
WORKDIR /home/pentester

# Configurar herramientas específicas
RUN mkdir -p /home/pentester/tools

# Comando por defecto
CMD ["/bin/bash"]

5.3 Construir Imagen

# Construir imagen desde Dockerfile
docker build -t pentesting-tools:latest .

# Construir con tag específico
docker build -t usuario/pentesting-tools:v1.0 .

# Construir con contexto diferente
docker build -t mi-imagen -f /ruta/Dockerfile /ruta/contexto

# Ver historial de build
docker history mi-imagen

6. Docker Compose

6.1 Introducción a Docker Compose

Docker Compose permite definir y ejecutar aplicaciones multi-contenedor con un solo archivo YAML.

6.2 Estructura Básica de docker-compose.yml

version: '3.8'

services:
  web:
    image: nginx:latest
    ports:
      - "80:80"
    volumes:
      - ./html:/usr/share/nginx/html
    networks:
      - app-network

  db:
    image: mysql:8.0
    environment:
      MYSQL_ROOT_PASSWORD: rootpass
      MYSQL_DATABASE: appdb
    volumes:
      - db-data:/var/lib/mysql
    networks:
      - app-network

volumes:
  db-data:

networks:
  app-network:
    driver: bridge

6.3 Ejemplo: Laboratorio de Pentesting con Docker Compose

version: '3.8'

services:
  # Servidor web vulnerable
  web-vulnerable:
    image: vulhub/webgoat:latest
    ports:
      - "8080:8080"
    networks:
      - pentest-network

  # Base de datos vulnerable
  db-vulnerable:
    image: vulnerables/cve-2017-9805
    ports:
      - "8081:8080"
    networks:
      - pentest-network

  # Herramientas de ataque
  attacker:
    image: kalilinux/kali-rolling:latest
    command: sleep infinity
    networks:
      - pentest-network
    volumes:
      - ./tools:/tools

  # Monitor de red
  wireshark:
    image: lscr.io/linuxserver/wireshark:latest
    container_name: wireshark
    environment:
      - PUID=1000
      - PGID=1000
    volumes:
      - /tmp/.X11-unix:/tmp/.X11-unix
    cap_add:
      - NET_ADMIN
      - SYS_ADMIN
    network_mode: host
    devices:
      - /dev/dri
    privileged: true

networks:
  pentest-network:
    driver: bridge

6.4 Comandos Docker Compose

# Iniciar servicios
docker-compose up -d

# Detener servicios
docker-compose down

# Ver logs
docker-compose logs -f

# Escalar servicios
docker-compose up -d --scale web=3

# Ejecutar comando en servicio específico
docker-compose exec web /bin/bash

7. Seguridad en Contenedores

7.1 Buenas Prácticas de Seguridad

# 1. No ejecutar como root
docker run -u 1000:1000 -it ubuntu:22.04

# 2. Limitar recursos
docker run --memory=512m --cpus=1.0 ubuntu:22.04

# 3. Usar red aislada
docker network create --internal pentest-net
docker run --network=pentest-net ubuntu:22.04

# 4. Escanear vulnerabilidades
docker scan mi-imagen

# 5. Actualizar contenedores regularmente
docker pull mi-imagen:latest
docker stop contenedor && docker rm contenedor
docker run -d --name contenedor mi-imagen:latest

7.2 Auditoría de Contenedores

# Ver detalles de contenedor
docker inspect <container_id>

# Ver procesos en contenedor
docker top <container_id>

# Ver uso de recursos
docker stats <container_id>

# Analizar capas de imagen
docker history mi-imagen

8.3 Script de Automatización

#!/bin/bash
# deploy-lab.sh

# Variables
LAB_NAME="pentest-lab"
COMPOSE_FILE="docker-compose-pentest.yml"

# Función para iniciar laboratorio
start_lab() {
    echo "Iniciando laboratorio de pentesting..."
    docker-compose -f $COMPOSE_FILE up -d
    echo "Laboratorio iniciado. Accede a:"
    echo "  - WebGoat: http://localhost:8080"
    echo "  - Attacker: docker exec -it ${LAB_NAME}_attacker_1 /bin/bash"
}

# Función para detener laboratorio
stop_lab() {
    echo "Deteniendo laboratorio..."
    docker-compose -f $COMPOSE_FILE down
    echo "Laboratorio detenido."
}

# Función para reiniciar laboratorio
restart_lab() {
    stop_lab
    start_lab
}

# Menú principal
case "$1" in
    start)
        start_lab
        ;;
    stop)
        stop_lab
        ;;
    restart)
        restart_lab
        ;;
    *)
        echo "Uso: $0 {start|stop|restart}"
        exit 1
        ;;
esac

docker run -it --rm kalilinux/kali-rolling nmap -sV 192.168.1.1

docker run -it --rm --network host kalilinux/kali-rolling nmap -sn 192.168.1.0/24

# Ejecutar ZAP en contenedor
docker run -u zap -p 8080:8080 -p 8090:8090 \
  -v $(pwd)/zap-data:/zap/wrk:rw \
  owasp/zap2docker-stable zap.sh -daemon \
  -host 0.0.0.0 -port 8080 \
  -config api.key=secret


**Tipos de redes:**
- `bridge`: Red por defecto (aislada)
- `host`: Usa red del host
- `none`: Sin red
- `overlay`: Para clústeres (Swarm)
- `macvlan`: Asigna MAC address al contenedor

### 4.2 Volúmenes y Directorios Montados

```bash
# Montar directorio del host en contenedor
docker run -it -v /ruta/host:/ruta/contenedor ubuntu:22.04

# Montar volumen con nombre
docker volume create mi-volumen
docker run -it -v mi-volumen:/ruta/contenedor ubuntu:22.04

# Montar como solo lectura
docker run -it -v /ruta/host:/ruta/contenedor:ro ubuntu:22.04

# Copiar archivos entre host y contenedor
docker cp /ruta/host/archivo.txt <container_id>:/ruta/contenedor/
docker cp <container_id>:/ruta/contenedor/archivo.txt /ruta/host/
```

## 5. Dockerfile: Creación de Imágenes Personalizadas

### 5.1 Estructura Básica de Dockerfile

```dockerfile
# Imagen base
FROM kalilinux/kali-rolling:latest

# Maintainer (obsoleto, usar LABEL)
LABEL maintainer="diego@ejemplo.com"

# Variables de entorno
ENV DEBIAN_FRONTEND=noninteractive

# Actualizar e instalar paquetes
RUN apt update && apt install -y \
    nmap \
    nikto \
    sqlmap \
    && rm -rf /var/lib/apt/lists/*

# Directorio de trabajo
WORKDIR /app

# Copiar archivos locales al contenedor
COPY . /app

# Exponer puertos
EXPOSE 80 443 22

# Comando por defecto al iniciar
CMD ["/bin/bash"]
```

### 5.2 Ejemplo: Imagen de Pentesting

```dockerfile
# Dockerfile para entorno de pentesting
FROM kalilinux/kali-rolling:latest

# Configurar entorno
ENV DEBIAN_FRONTEND=noninteractive
ENV LANG=C.UTF-8

# Actualizar sistema
RUN apt update && apt upgrade -y

# Instalar herramientas de pentesting
RUN apt install -y \
    # Herramientas de enumeración
    nmap \
    enum4linux \
    ldap-utils \
    # Herramientas web
    nikto \
    dirb \
    dirbuster \
    wfuzz \
    sqlmap \
    # Herramientas de explotación
    metasploit-framework \
    exploitdb \
    # Herramientas de post-explotación \
    mimikatz \
    # Herramientas de red \
    wireshark \
    tcpdump \
    netcat-traditional \
    # Herramientas de fuerza bruta \
    hydra \
    john \
    # Limpieza \
    && rm -rf /var/lib/apt/lists/*

RUN useradd -m -s /bin/bash pentester

```

## 6. Docker Compose

### 6.1 Introducción a Docker Compose

Docker Compose permite definir y ejecutar aplicaciones multi-contenedor con un solo archivo YAML.

### 6.2 Estructura Básica de docker-compose.yml

```yaml
version: '3.8'

services:
  web:
    image: nginx:latest
    ports:
      - "80:80"
    volumes:
      - ./html:/usr/share/nginx/html
    networks:
      - app-network

  db:
    image: mysql:8.0
    environment:
      MYSQL_ROOT_PASSWORD: rootpass
      MYSQL_DATABASE: appdb
    volumes:
      - db-data:/var/lib/mysql
    networks:
      - app-network

volumes:
  db-data:

networks:
  app-network:
    driver: bridge
```

### 6.3 Ejemplo: Laboratorio de Pentesting con Docker Compose

```yaml
version: '3.8'

services:
  # Servidor web vulnerable
  web-vulnerable:
    image: vulhub/webgoat:latest
    ports:
      - "8080:8080"
    networks:
      - pentest-network

  # Base de datos vulnerable
  db-vulnerable:
    image: vulnerables/cve-2017-9805
    ports:
      - "8081:8080"
    networks:
      - pentest-network

  # Herramientas de ataque
  attacker:
    image: kalilinux/kali-rolling:latest
    command: sleep infinity
    networks:
      - pentest-network
    volumes:
      - ./tools:/tools

  # Monitor de red
  wireshark:
    image: lscr.io/linuxserver/wireshark:latest
    container_name: wireshark
    environment:
      - PUID=1000
      - PGID=1000
    volumes:
      - /tmp/.X11-unix:/tmp/.X11-unix
    cap_add:
      - NET_ADMIN
      - SYS_ADMIN
    network_mode: host
    devices:
      - /dev/dri
    privileged: true

networks:
  pentest-network:
    driver: bridge
```

### 6.4 Comandos Docker Compose

```bash
# Iniciar servicios
docker-compose up -d

# Detener servicios
docker-compose down

# Ver logs
docker-compose logs -f

# Escalar servicios
docker-compose up -d --scale web=3

# Ejecutar comando en servicio específico
docker-compose exec web /bin/bash
```

### 7.1 Buenas Prácticas de Seguridad

### 8.3 Script de Automatización

```bash
#!/bin/bash
# deploy-lab.sh

# Variables
LAB_NAME="pentest-lab"
COMPOSE_FILE="docker-compose-pentest.yml"

# Función para iniciar laboratorio
start_lab() {
    echo "Iniciando laboratorio de pentesting..."
    docker-compose -f $COMPOSE_FILE up -d
    echo "Laboratorio iniciado. Accede a:"
    echo "  - WebGoat: http://localhost:8080"
    echo "  - Attacker: docker exec -it ${LAB_NAME}_attacker_1 /bin/bash"
}

# Función para detener laboratorio
stop_lab() {
    echo "Deteniendo laboratorio..."
    docker-compose -f $COMPOSE_FILE down
    echo "Laboratorio detenido."
}

# Función para reiniciar laboratorio
restart_lab() {
    stop_lab
    start_lab
}

# Menú principal
case "$1" in
    start)
        start_lab
        ;;
    stop)
        stop_lab
        ;;
    restart)
        restart_lab
        ;;
    *)
        echo "Uso: $0 {start|stop|restart}"
        exit 1
        ;;
esac
```
